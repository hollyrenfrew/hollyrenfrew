Python RPG Adventure Game

A text-based Python game originally developed for IT-140 and enhanced for CS-499 to include persistent player data using SQLite. This project serves as the Databases artifact in my capstone portfolio.

Original Functionality

  • Exploration through typed commands (N, S, E, W)
  • Combat system with player stats, weapons, and enemies
  • Room-based storyline with modular Python files
  • All data stored in memory (no saving/loading)

Enhancements for CS-499

  • Integrated SQLite database for persistent save/load
  • Created tables for player, rooms, inventory, and enemies
  • Added data access layer (`database_service.py`)
  • Refactored logic to support long-term progression
  • Sanitized user input to protect the database
  • Improved code comments and documentation

Skills Demonstrated

  • Database schema design and normalization
  • Secure input handling and defensive programming
  • Refactoring legacy code for scalability and maintainability
  • Persistent storage using Python and SQL

Course Outcomes Met

  • Outcome 3: Design and evaluate computing solutions (database schema, algorithm refactor)
  • Outcome 4: Use well-founded techniques/tools (SQLite, DAO pattern)
  • Outcome 5: Security mindset (sanitization, structure, defensive programming)
